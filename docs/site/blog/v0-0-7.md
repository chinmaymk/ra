# ra 0.0.7: Subscriptions, Shell Tools, and Hot-Reload

<span class="blog-date">April 6, 2026</span>

0.0.7 is the biggest ra release yet. Two new providers let you use your existing ChatGPT and Claude subscriptions instead of API keys. Custom tools can now be shell scripts in any language. Config hot-reload means you never restart the agent during development. And a new recipe ships a batteries-included agent with 16 skills and 8 middleware hooks.

Here's everything that's new.

## Use your subscriptions

The most-requested feature: run ra without API keys. Two new providers make this possible.

### Codex provider (ChatGPT Plus/Pro)

The `codex` provider routes requests through OpenAI's Codex backend using your ChatGPT subscription credits. No per-token billing ā€” usage draws from your existing plan quota.

```bash
ra login codex          # browser-based OAuth
ra --provider codex --model gpt-5.4 "Hello"
```

Plus subscribers get access to `gpt-5.4` and `o4-mini`. Pro subscribers also get `gpt-5.3-codex` and `o3`. Device code flow (`--device-code`) is supported for headless environments like SSH and Docker.

### Anthropic Agents SDK provider (Claude Pro/Max)

The `anthropic-agents-sdk` provider wraps the Claude CLI to use your Claude subscription. Each turn spawns a subprocess, serializes ra's conversation as XML, and returns control after one model call. ra owns the loop, context, and tools ā€” the SDK just provides the model.

```bash
ra --provider anthropic-agents-sdk "Hello"
```

No API key needed. Just install the Claude CLI and authenticate with `claude login`. Extended thinking works too.

### ra login

A new `ra login` command handles authentication flows for subscription providers:

```bash
ra login codex           # ChatGPT OAuth (PKCE)
ra login codex --device-code  # headless
ra login claude          # Claude CLI auth
```

## Shell script tools

Custom tools are no longer TypeScript-only. Any shell script or executable can be a tool. Scripts with known extensions (`.sh`, `.py`, `.rb`, `.pl`, `.lua`, and more) are auto-detected:

```yaml
agent:
  tools:
    custom:
      - "./tools/health-check.sh"
      - "./tools/lint.py"
      - "shell: /usr/local/bin/my-tool --strict"
```

Shell tools self-describe via a `--describe` flag that outputs JSON, and receive input on stdin:

```bash
#!/bin/bash
if [ "$1" = "--describe" ]; then
  echo '{"name":"HealthCheck","description":"Check endpoint health","parameters":{"url":{"type":"string"}}}'
  exit 0
fi
read -r input
url=$(echo "$input" | jq -r '.url')
curl -s -o /dev/null -w "HTTP %{http_code}" "$url"
```

This same protocol powers shell middleware ā€” write a Python script, a Ruby script, or a compiled binary, and it works as both a tool and a middleware hook.

## Config hot-reload

ra now watches your config file and everything it references ā€” system prompts, middleware scripts, custom tools, context resolvers ā€” and reloads them automatically between agent loops.

```bash
# Terminal 1: start the agent
ra

# Terminal 2: edit middleware while the agent is running
vim middleware/audit.ts
```

The next message picks up the changes. No restart. Provider connections rebuild if credentials change. Module caches clear so you always get fresh code. Long-lived state (sessions, MCP connections, memory) is preserved.

Hot-reload is on by default. Disable it with `agent.hotReload: false` if you don't need it.

## oh-my-ra recipe

We shipped [oh-my-ra](/blog/oh-my-ra) ā€” a complete recipe inspired by oh-my-claude and oh-my-codex, built natively on ra's primitives. 16 skills, 8 middleware hooks, 2 custom tools, and 3 auto-run scripts. Read the [full post](/blog/oh-my-ra) for details.

Highlights:
- **Workflow skills** like `/ultrawork`, `/deep-research`, `/team`, and `/pair`
- **Engineering skills** like `/debugger`, `/architect`, `/refactor`, and `/migrate`
- **Quality middleware** that blocks destructive commands, detects loops, and tracks context usage
- **ProjectScan** and **DependencyCheck** custom tools

```bash
bun run ra --config recipes/oh-my-ra/ra.config.yaml
```

## auto-improve recipe

A new recipe that runs a parallel self-improvement loop. The agent evaluates its own config, skills, and middleware ā€” then proposes and applies improvements. Combined with hot-reload, changes take effect on the next iteration without restarting.

## Config validation

ra now validates your entire config at startup and reports clear, actionable errors. Typos in provider names, invalid middleware paths, malformed permission rules ā€” all caught before the agent starts instead of failing silently at runtime.

## Better tool formatting

Tool calls and results now render with specialized formatters in the REPL. File contents show syntax-highlighted previews. Bash commands show the command being run. Diffs show colored additions and deletions. The animated spinner is replaced with a static indicator, and tool results update in-place instead of scrolling the terminal.

## Context compaction improvements

Compaction now extracts metadata ā€” active plans, key decisions, file paths, error patterns ā€” and weaves them into enriched summaries. This means compacted conversations retain more useful context with fewer tokens. Combined with oh-my-ra's context-guard middleware that warns at 70% and alerts at 85% usage, context management is now largely automatic.

## Conversation history handling

Tool result continuations are now handled correctly when resuming sessions. Previously, resuming a session mid-tool-call could produce malformed message sequences. The history handler now detects and repairs these edge cases.
